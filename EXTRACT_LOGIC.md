# extract_problems.py — 로직 설계 문서

## 개요

수학 모의고사 PDF(시험지 + 해설)에서 문제별 이미지를 추출한다.
출력: `public/problems/{YY}/{examName}_{num:02d}.png` (문제), `..._sol.png` (해설)

---

## 입력 구조

```
asset/{YY}/
  {월}.pdf          # 시험지
  {월}해설.pdf      # 해설 (선택)
```

---

## 시험지 PDF 구조

- **2컬럼** 레이아웃 (가로가 긴 A3 용지, 세로로 2열)
- 각 문제는 `"N."` 으로 시작하는 텍스트 블록이 있음
- 번호는 텍스트 레이어에서 추출 가능
- 문제 수: 30개 (1~30)

---

## 해설 PDF 구조

### 페이지 구성
```
page 0:   [빠른정답 표] + [1번 해설 시작] + [2번 해설] + ...
page 1~N: [계속되는 해설들]
page X:   [선택과목명(확률과통계/미적분/기하)] + [빠른정답 표] + [23번 해설] + ...
page X+1: [계속]
...
```

> **중요**: page 0을 통째로 skip하면 안 됨. 빠른정답 표 아래에 바로 해설이 시작됨.

### 페이지 레이아웃

| 항목 | 내용 |
|------|------|
| 상단 머리말 | 시험명, 과목명 등 (앵커 탐지 시 제외 필요) |
| 하단 꼬리말 | 페이지 번호 등 (앵커 탐지 시 제외 필요) |
| 컬럼 수 | **2열** 또는 **3열** (PDF마다 다름, 자동 감지 필요) |

### 해설 형식 (2종류)

**타입 A** (완전 형식)
```
N. 출제의도 : ~~~
   정답풀이 : ~~~
   정답 : ~~~
```

**타입 B** (축약 형식)
```
N. 출제의도 : ~~~
```
정답풀이·정답 멘트 없이 풀이 내용만 있는 경우.

> 공통 앵커: **"출제의도"** — 두 타입 모두 포함. 이것으로 문제 시작점 탐지.

### 빠른정답 구간 (앵커 없음)
- 첫 페이지 상단의 공통 빠른정답 표
- 선택과목 시작 시 해당 과목명 + 빠른정답 표
- **"출제의도" 텍스트가 없으므로 자동으로 제외됨**

### 수능 특이사항
- 공통: 1~22번
- 선택과목 3개(확통/미적/기하)가 각각 23~30번 해설을 가짐
- → 23~30번 해설이 3세트 존재 (처리 방침 미정)

## 컬럼 레이아웃

| 구분 | 시험지 | 해설 |
|------|--------|------|
| 컬럼 수 | 2열 고정 (A3) | **2열 또는 3열** (PDF마다 다름) |
| 컬럼 구분선 | (확인 필요) | 3열: **수직 실선 있음** / 2열: **실선 없음** |
| 컬럼 감지 방법 | 수직 실선 감지 우선, 없으면 앵커 x 클러스터링 | 동일 — **실선 유무로 2열/3열 판별 가능** |
| 읽기 순서 | 왼쪽→오른쪽 컬럼 순 | 동일 |

### 컬럼 감지 로직 (현재 `find_col_dividers` + `detect_n_cols_from_dividers`)
```
get_drawings()로 수직 실선 탐지
  → 실선 1개: 2열 구분 → n_cols = 2   (실제로는 실선 없음이므로 해당 없음)
  → 실선 2개: 3열 구분 → n_cols = 3
  → 실선 없음: 2열로 간주 (앵커 x좌표로 보정)
```

> **핵심**: 해설에서 수직 실선이 있으면 반드시 3열, 없으면 2열.
> 현재 코드에서 이미 `get_drawings()`로 수직 실선을 감지하므로 그대로 활용 가능.

## 머리말 / 꼬리말 처리

| 구분 | 가로 실선 있는 경우 | 가로 실선 없는 경우 |
|------|-------------------|-------------------|
| 감지 방법 | `get_drawings()`로 가로 실선 y좌표 탐지 | 페이지 비율 추정 (현재 상하 5%) |
| 머리말 하단 y | 상단 가로 실선 y1 | page_h × 0.10~0.15 추정 |
| 꼬리말 상단 y | 하단 가로 실선 y0 | page_h × 0.88~0.90 추정 |

### 예정 로직
```python
def get_content_bounds(page):
    """머리말/꼬리말을 제외한 실제 내용 영역 (y_top, y_bottom)"""
    page_h = page.rect.height
    h_lines = find_horizontal_lines(page)  # 가로 실선 y좌표 목록

    # 상단 실선: 머리말 경계
    top_lines = [y for y in h_lines if y < page_h * 0.3]
    y_top = max(top_lines) if top_lines else page_h * 0.12

    # 하단 실선: 꼬리말 경계
    bot_lines = [y for y in h_lines if y > page_h * 0.7]
    y_bottom = min(bot_lines) if bot_lines else page_h * 0.88

    return y_top, y_bottom
```

이 함수로 앵커 탐지 범위와 렌더링 크롭 영역을 동적으로 결정.

## 앵커 탐지 전략

### 시험지: "N." 기반

```python
PROBLEM_RE = re.compile(r'^(\d+)\.')

for block in page.get_text('blocks'):
    lines = block.text.split('\n')
    for line in lines:
        if PROBLEM_RE.match(line):
            # → 앵커 등록 (num, page, col, y)
            break
```

**빠른정답 제거 필터**
1. 같은 (page, num) 쌍에서 `internal=True`(블록 첫줄이 아닌 곳 매치) 우선
2. 컬럼별 y-간격 그룹에서 번호 점프 ≥ 3이면 제거
3. 같은 번호가 페이지에 3회 이상 → 해당 페이지 전체 제거

### 해설: "출제의도" 기반 ← **미구현 (예정)**

```python
for block in page.get_text('blocks'):
    if '출제의도' in block.text:
        num = extract_num_from_block(block)   # 같은 블록 또는 직전 블록에서 추출
        if num is None:
            num = infer_num_by_position(...)  # 앞뒤 번호 사이로 유추
        # → 앵커 등록 (num, page, col, y)
```

**빠른정답 자동 제외**: 빠른정답 구간에는 "출제의도"가 없으므로 별도 필터 불필요.

**번호 추출 실패 시 유추**
- 앞 앵커 번호 A, 뒤 앵커 번호 B 사이에 위치한다면 → A+1 ~ B-1 사이에서 순서로 배정

## 영역 계산 (build_problem_regions)

각 앵커로부터 렌더링 영역(rect)을 계산.

```
문제 N의 영역:
  y_start = anchor_N.y - prob_top_margin
  y_end   = anchor_{N+1}.y - prob_gap  (다음 문제 시작 바로 위)
  x_range = 해당 컬럼의 x 범위
```

### 다중 세그먼트 (해설이 여러 열/페이지에 걸칠 때)

```
curr_pos = curr.page * n_cols + curr.col   (읽기 순서 위치값)
end_pos  = next.page * n_cols + next.col

if end_pos - curr_pos <= MAX_SPAN(=3):
    각 중간 컬럼 순회
    → 중간 컬럼에 앵커 있거나 실제 내용(col_has_content) 있으면 stop_early
    → stop_early면 단일 세그먼트로 폴백
    → 아니면 각 컬럼 조각을 세그먼트 목록에 추가
else:
    단일 세그먼트 폴백
```

**세그먼트 y 범위**

| 위치 | y_start | y_end |
|------|---------|-------|
| 첫 세그먼트 | anchor.y - prob_top | 페이지 하단 - crop |
| 중간 세그먼트 | find_col_content_start() | 페이지 하단 - crop |
| 끝 세그먼트 | find_col_content_start() | next_anchor.y - gap |

`find_col_content_start()`: 페이지 상단 15% 헤더 건너뛰고 해당 컬럼의 첫 텍스트 블록 y 반환.

### 패턴 정리

**용지 × 열 수 관계**
- A3 (841×1190): 항상 **3열**, 수직선 x ≈ [310, 530]
- A4 (595×841): **2열** 또는 **3열** (4월 시험은 A4 3열, 수직선 x ≈ [220, 380])
- B4? (612×791): 3열, 수직선 x ≈ [190, 340] — 25년 5월만 해당

**머리말 가로선**
- 대부분 16~24% 위치에 존재
- 꼬리말은 76~92% (없는 파일도 다수)

**"출제의도" 탐지 실패**
- 22수능해설 (21/22 수능해설.pdf): ❌ — 해설 포맷 자체가 다름
- 24수능해설 (23/24 수능해설.pdf): ❌ — 동일 원인 추정

**수직선 오탐**
- 23/7월해설: 3열인데 5열로 감지됨 → 수직선이 쌍으로 그어져 있음 (310+320, 520+530) → 감지 로직에서 근접 중복 제거 필요 (threshold ~20pt)
